# Brain — контекстный мост для Gemini

Когда видишь `@brain <тема>` — определи тему:

- Если тема **install**, **setup** или **установка** → запусти установку
- Иначе → запусти explore-агента для генерации контекста

## Установка

Пользователь хочет установить brain глобально. Выполни:

```
node install.js
```

Команда запускается из корня `brain-pack/` (текущая рабочая директория). Скрипт сам запросит подтверждение перед каждым шагом.

Можно также предложить опции:
- `node install.js -y` — без подтверждения (для автоматизации)
- `node install.js --dry-run` — только показать план, ничего не делать

После завершения напиши пользователю: "Brain установлен глобально. Используй `@brain <тема>` в любом проекте."

## Генерация контекста

Запусти explore-агента через `task` tool, передав ему тему и следующие инструкции:

### Инструкция для explore-агента

Ты — контекстный мост между проектом и Google Gemini AI.

Пользователь хочет обсудить с Gemini тему: $ARGUMENTS

1. Проанализируй тему — выдели ключевые сущности.
2. Найди в проекте файлы, связанные с темой. Игнорируй нерелевантные (стили, картинки, платёжные модули).
3. Создай файл `gemini_context.txt` в корне проекта со структурой:

```
=== ПРОМПТ ===
{тема пользователя}

=== КОНТЕКСТ ===

File: path/to/file (строки X-Y)
Что делает файл, почему релевантен

{выжимка: функции, типы, роуты, схемы — 50-70 строк}
...
```

4. Запусти генерацию PDF: `gemini-pdf gemini_context.txt gemini_task.pdf`
5. Удали `gemini_context.txt`.
6. Напиши пользователю: "PDF готов: gemini_task.pdf (N KB). Загрузи в поиск Google AI и обсуждай."

Верни вызвавшему агенту отчёт: какие файлы найдены, сколько строк, размер PDF, путь.
